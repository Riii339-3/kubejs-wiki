# 基礎知識

KubeJSで好きなものを作る前に、まずは最低限覚えておくべき情報を書いておきます。

## Tickについて

マインクラフトでは、主に`tick`という単位で動いてます。  
1tick=約0.05、つまり20tickで約1秒です。ただし、これはTPSが20であることが前提です。  

## サーバーとクライアントについて

マインクラフトは、v1.7以降シングルプレイであってもサーバーとクライアントが分離しています。  
サーバー側では主にチャンク生成、かまどなどのレシピ処理などを行っています。  
逆にクライアント側では主にレンダリングを行っています。  
また、アイテムやブロックの登録は双方に必要です。  

## ItemとItemStack、EntityTypeとEntityについて

Moddingをやったことのある方ならご存知でしょう。ItemとItemStackについてです。  
主に**登録するデータをItem**、**ゲーム内で実態として登場するのがItemStack**となります。

## JavaScriptについて

KubeJSは名前の通り`JavaScript`という言語で記載します。  
そのため、`for`文や`if`文などを多用する場面もあります。  
わからなくなったらJavaScriptに関するブログやドキュメントを読んでみるといいでしょう。  
ただし、KubeJSのエンジンは**Rhino**です。少し古いので最新の記載をするとエラーが発生する可能性もあります。あとTypeScriptに対応してない。  

## 3つのスクリプト

初めてKubeJS入りマイクラを起動した際、インスタンスフォルダに`kubejs`というフォルダが生成されています。この中を除くと`assets`、`data`、`server_scripts`、`startup_scripts`、`client_script`…などが生成されていると思います。  
assets、dataは主にリソパ・データパック関連ですのでおいておいて、ここでは3つのスクリプトについて説明します。

- StartupScripts: 
主にアイテム/ブロックの登録・編集などの**サーバー、クライアント双方が必須とする**ものを書きます。  
- ServerScripts: 
主にレシピの作成・編集など**サーバーサイドで必須**なものを書きます。  
- ClientScrips: 
主に…何に使うんでしょう？**クライアントサイドで必須**なものを書きます。  

基本的に、当wikiでは記載されているスクリプトをどこに置けばをすべて書いておりますので、それに従って書いてください。

## 便利なコマンド

ゲーム内で`/kubejs`または`/kjs`と入力すると、いろいろ出てくると思います。これらが、KubeJSで追加されるコマンドです。  
主に`/kjs hand`は多用します。メインハンドにアイテムを持って実行すると、そのアイテムidや紐づけられたブロックのid、そしてタグなどが閲覧できます。さらに、チャット上から該当のidをクリックすることでそのidをコピーすることができます。  

## Windowsパスについて
ファイルやフォルダの場所を示す際、`./server_scripts/example.js`のような表記をしているのを当Wikiではたくさん見ることになるでしょう。この表記方法は**Windows パス**と呼ばれるものです。  
最初は驚くかもしれませんが、慣れるのはとても簡単です。`/`はフォルダを介したことを表しています。  
上記の場合、`kubejs`フォルダの中にある`server_scripts`フォルダの中の`example.js`となります。ね、難しくないでしょ？  
では最初の`./`とは何なのか。これは**今いるフォルダの場所**になります。上記の場合、`kubejs/`が該当します。KubeJSなんて上記に書いてない？って方はこのページにある3つのスクリプトを読み返してください。そこには`kubejs`フォルダがあるでしょう。もし`kubejs`を含めて書くのであれば、`kubejs/server_scripts/example.js`となります。長いですね、嫌ですね。なので、当Wikiでは特に明記されてない限り`./`は`kubejs/`を表します。  

さて、当Wikiではあまり見かけないかもしれませんが、もう一つわかりずらい表し方があります。`../`です。こちらは、`./`のさらに親である、`minecraft/`となります。ややこしい。  
ですがもし、インスタンスフォルダから全文書こうとすると`minecraft/kubejs/server_scripts/example.js`となってしまいます。とっても長いですね、とっても嫌ですね。なので、当Wikiでは**通常の親フォルダを`Kubejs`**として扱います。
