# 基礎知識

KubeJSで好きなものを作る前に、まずは最低限覚えておくべき情報を書いておきます。

## Tickについて

マインクラフトでは、主に`tick`という単位で動いてます。  
1tick=約0.05、つまり20tickで約1秒です。ただし、これはTPSが20であることが前提です。  

## サーバーとクライアントについて

マインクラフトは、v1.7以降シングルプレイであってもサーバーとクライアントが分離しています。  
サーバー側では主にチャンク生成、かまどなどのレシピ処理などを行っています。  
逆にクライアント側では主にレンダリングを行っています。  
また、アイテムやブロックの登録は双方に必要です。  

## ItemとItemStack、EntityTypeとEntityについて

Moddingをやったことのある方ならご存知でしょう。ItemとItemStackについてです。  
主に**登録するデータをItem**、**ゲーム内で実態として登場するのがItemStack**となります。

## levelについて

鯖管理者は一度は見たことであろう、level.dat。大体それ。  
levelは**ディメンジョン内の全データ**を指します。  
KubeJSでは大体eventから取得しますが、何らかの理由で独自に取得したい場合levelから取得することがほとんどです。~~大体levelで解決する。~~  

## JavaScriptについて

KubeJSは名前の通り`JavaScript`という言語で記載します。  
そのため、`for`文や`if`文などを多用する場面もあります。  
わからなくなったらJavaScriptに関するブログやドキュメントを読んでみるといいでしょう。  
ただし、KubeJSのエンジンは**Rhino**です。少し古いので最新の記載をするとエラーが発生する可能性もあります。あとTypeScriptに対応してない。  

## 3つのスクリプト

初めてKubeJS入りマイクラを起動した際、インスタンスフォルダに`kubejs`というフォルダが生成されています。この中を除くと`assets`、`data`、`server_scripts`、`startup_scripts`、`client_script`…などが生成されていると思います。  
assets、dataは主にリソパ・データパック関連ですのでおいておいて、ここでは3つのスクリプトについて説明します。

- StartupScripts: 
主にアイテム/ブロックの登録・編集などの**サーバー、クライアント双方が必須とする**ものを書きます。  
- ServerScripts: 
主にレシピの作成・編集など**サーバーサイドで必須**なものを書きます。  
- ClientScrips: 
主に…何に使うんでしょう？**クライアントサイドで必須**なものを書きます。
